# Marketing Platform

# Purpose

The marketing layer translates platform infrastructure into:

- customer value
- selling points
- onboarding flows
- investor narratives
- landing pages
- pricing explanations
- educational content

---

# Core Principle

Every feature should become:

- a customer benefit
- a pain-solver
- a landing page section
- a product story
- a pricing justification

---

# Planned Areas

| Area | Purpose |
|---|---|
| Landing Pages | product positioning |
| Feature Pages | individual feature marketing |
| Industry Pages | PMA, family office, enterprise |
| Docs | onboarding and education |
| Comparisons | full vs cloud AI |
| Pricing | plans and deployment tiers |
| Security | trust and governance |
| Hardware | Mac + Thor positioning |

# Strategic Goal

The marketing layer should eventually generate:

- SEO pages
- launch notes
- social content
- investor summaries
- feature highlights
- onboarding copy
- sales enablement

from the same feature PRDs used internally.
