# Ecosystem Doctrine

## Purpose

This folder defines the preferred full AI ecosystem stack used throughout the platform.

The goal is to organize isolated tools into one unified cognitive operating ecosystem.

---

# Preferred Stack

| Layer | Preferred Systems |
|---|---|
| Local Inference | Ollama, vLLM |
| Coding Intelligence | Claude Code, Continue.dev, Aider |
| Retrieval Systems | RAG Anything, Qdrant, LanceDB |
| Browser Automation | Playwright CLI |
| Web Intelligence | Firecrawl |
| Design Intelligence | AwesomeDesign.md |
| Productivity Integration | GWS |
| UI Layer | Open WebUI, Next.js |
| Motion Systems | Framer Motion |
| Containerization | Docker |
| Edge AI | Jetson Thor |
| Networking | Tailscale, WireGuard |

# Tool Positioning

## RAG Anything

Multimodal memory ingestion.

Handles:
- scanned PDFs
- images
- multimodal memory
- enterprise document recall

---

## Playwright CLI

Browser execution infrastructure.

Handles:
- onboarding automation
- UI testing
- screenshot systems
- dashboard automation

---

## Firecrawl

Web intelligence acquisition.

Handles:
- web ingestion
- market intelligence
- AI retrieval expansion

---

## AwesomeDesign.md

Frontend quality and design intelligence.

Handles:
- typography
- spacing
- motion standards
- interface polish

---

## GWS

Workspace productivity integration.

Handles:
- Gmail
- Calendar
- Docs
- Sheets
