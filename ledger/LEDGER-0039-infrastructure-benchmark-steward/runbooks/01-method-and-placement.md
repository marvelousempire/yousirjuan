# Method and placement sequence

1. Resolve reported identity separately from transport names and IP addresses.
2. Select a suite that represents the actual task or service.
3. Pin engine, Git SHA, fixture, runtime, repetitions, concurrency, cache, target, and route.
4. Run immutable receipts and retain calibration failures.
5. Reject comparisons whose method keys differ.
6. Evaluate latency, throughput, quality, interference, capacity, availability, recovery, and cost.
7. Promote per workload only; record review-required when gates are incomplete.
8. Project the decision into DustPan and Report Card without moving raw evidence.

External tools are adapters, not replacements. Speedtest measures WAN path. Geekbench measures
synthetic CPU/GPU. Disk Drill measures recovery/health. Benchlab measures the family's declared
infrastructure workloads and owns certification receipts.
